# AGENTS.md

## プロジェクト概要

このプロジェクトは以下の技術構成で作成された REST API アプリケーションです。

- Spring Boot
- MyBatis
- PostgreSQL

JANコードを利用した商品・価格検索APIを提供します。

---

## 設計方針

- 業務ロジックは Service 層へ集約する
- Mapper は DBアクセスのみを担当する
- Controller は薄く保つ
- DTO は必要に応じて Java record を利用する
- 過剰な抽象化を避ける
- 可読性を重視する

---

## DB設計方針

- PostgreSQL を前提とする
- MyBatis XML はシンプルで読みやすく保つ
- Trigger や DB制約は必要最小限とする
- 過度に複雑な動的SQLを避ける
- schema.sql は保守しやすい構成にする

---

## エラーハンドリング方針

- 不要な try-catch を増やさない
- 一時的な debug コードや log 出力を残さない
- 例外を握りつぶさない
- 表面的なエラーではなく root cause を修正する

---

## テスト方針

- テストを無効化して問題を回避しない
- assertion を弱めて無理やり通さない
- テストコードは可読性を重視する
- テストのためだけに本番コードを不自然に変更しない
- 原因調査後に最小限の修正を行う

---

## 修正ルール

- 必要最小限の変更を行う
- 依頼範囲外のリファクタを行わない
- 既存コードを残したまま追記しない
- 重複コードを作らない
- 調査コードを最終成果物に残さない

---

## コーディング方針

- 可読性を優先する
- メソッドは単一責務を意識する
- 過剰設計を避ける
- 既存プロジェクトの命名規則・構成に合わせる

---

## デバッグ時のルール

修正前に以下を実施すること。

1. root cause を特定する
2. 原因を簡潔に説明する
3. 最小限の修正を行う
4. テスト結果を確認する

推測による大規模修正は禁止。
